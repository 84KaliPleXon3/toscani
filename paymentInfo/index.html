Notes

  • Right now, the two-digit year only allows for 12-29. Which is fine, but there's no support for automatically updating this when the year changes. We would have to update the mask manually. Might be better to leave it alone and allow server/client validation to handle this case in production.
  • On iOS, you need to have the meta tag for mobile app capable if you want the JavaScript focus events to work. Not sure this workaround applies to Android (I'm guessing no). Did a lot of looking around, and this is a common problem. Couldn't find any direct Webkit bugs though. Not sure if this is intentional or a real bug.
